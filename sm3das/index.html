Next-Gen Porting

The Starshine Engine.

A compatibility layer built to run the entirety of Sunshine and Galaxy in WebAssembly — without emulation. Starshine translates original engine behavior into a native browser runtime, enabling near-native performance, full control support, and a unified system powering multiple 3D Mario titles.

🌊
Sunshine Compatibility
Recreates FLUDD physics, actor systems, and JSystem behavior to run Sunshine natively in the browser.
🌌
Galaxy Runtime
Supports Galaxy’s gravity, planet systems, and object-driven level architecture through a custom runtime layer.
⚡
No Emulation
Runs game logic directly in WebAssembly — no Dolphin, no interpreter slowdown, just raw execution.
🧠
Engine Translation
Converts original engine calls into modern Web APIs like WebGL and Web Audio.
🎮
Unified Controls
Shared input system across Sunshine and Galaxy with keyboard, mouse, and full gamepad support.
🚀
Multi-Game Engine
One engine powering multiple Mario titles — future-proof for Galaxy 2 and beyond.
Under the Hood

How it works.

Each game is a curated WebAssembly browser port — built on top of community decompilation projects and existing WASM builds, wrapped in a clean custom launcher shell with unified controls, save states, and a consistent UI across all four titles.

⚡
WebAssembly (WASM)
Game logic runs as near-native WASM bytecode — real performance, no emulation overhead.
🎨
WebGL Renderer
OpenGL calls translated to WebGL2 for hardware-accelerated 3D in any modern browser.
🎮
Gamepad API
Full controller support — plug in any gamepad and play with native console feel.
🔊
Web Audio API
Original game audio routed through the Web Audio API for authentic sound.
💾
Save States
Progress saved to IndexedDB — your game persists between sessions automatically.
🌐
GitHub Pages
Hosted on static GitHub Pages — zero server, zero cost, always on.
