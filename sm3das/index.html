Under the Hood

How it works.

Each game is compiled from its decompiled C source to WebAssembly using Emscripten, then wrapped in a custom HTML5 shell that handles input, audio, and rendering via WebGL. The result runs at full speed with no plugin or download.

⚡
WebAssembly (WASM)
Game logic compiled from C to near-native WASM bytecode via Emscripten.
🎨
WebGL Renderer
OpenGL calls translated to WebGL2 for hardware-accelerated 3D rendering.
🎮
Gamepad API
Full controller support — plug in any gamepad and play with native feel.
🔊
Web Audio API
Original audio streams routed through the Web Audio API for authentic sound.
📦
Decompiled Source
Built on the sm64, Sunshine, and Galaxy decompilation projects.
🌐
GitHub Pages
Hosted entirely on static GitHub Pages — zero server, zero cost, always on.

Development Roadmap

Full transparency on where each game is in the porting process.

Phase 01 — Complete
SM64 — Core Engine Compilation
Successfully compiled the sm64 decompilation project to WASM via Emscripten. Basic rendering and game loop running in browser.
Phase 02 — In Progress
SM64 — Input, Audio & Polish
Wiring keyboard + gamepad input, fixing audio streaming, implementing save states via IndexedDB, and building the custom HTML5 shell UI.
Phase 03 — Upcoming
SM64 — Public Beta Launch
Open beta of Super Mario 64 Web. Full playable from world 1 to Bowser. Performance tuning for mobile and lower-end devices.
Phase 04 — Planned Q3 2026
Sunshine — WASM Compilation
Begin porting the Sunshine decompilation. GameCube architecture is more complex — targeting Dolphin-compatible rendering pipeline via WebGL.
Phase 05 — Planned 2027
Galaxy — Wii Port Research
Galaxy's Wii architecture and motion controls make it the most ambitious port. Research phase to determine feasibility of WebGL + pointer events.
Phase 06 — The Dream
SM3DAS Web — Full Launch
All four games playable in the browser. Unified launcher. Save states. Gamepad support. The full All-Stars experience, browser-native.
