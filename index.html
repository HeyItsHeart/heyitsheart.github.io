About Me

Dev who ships.
Not just plans.

I build complete, production-grade apps as single HTML files — no frameworks, no build steps, no deployment complexity. Everything runs in the browser and works offline. If it's not shippable, it's not worth building.

My focus is on tools people actually use — from game library managers to browser-native WebAssembly engine ports. I care deeply about feel: animations, typography, interaction design, the tiny details that make something alive.

Currently deep in Game Vault (a full game selling platform in a single HTML file) and WebEngine — porting Half-Life's GoldSrc and Valve's Source Engine to WASM so they run in any browser tab.

How I Build

The philosophy
behind it all.

"If it needs a server, a build step, or a README to run — it's already too complicated. The best tools disappear. They just work."
01 —
Ship first, polish later
A working app with rough edges beats a perfect design that never ships. Get it live, then make it beautiful.
02 —
Single-file discipline
Constraints breed creativity. Everything in one file means every byte earns its place. No bloat, no dependencies, no excuses.
03 —
Feel matters as much as function
Animations aren't decoration. Typography isn't optional. The way something feels to use is part of how it works.
04 —
Own your stack
No framework lock-in. No npm. No cloud dependency. If I build it, I understand every line of it.
